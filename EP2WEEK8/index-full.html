Greedy Choice Property

Greedy Algorithm คือ การเลือกสิ่งที่ดีที่สุด... ณ วินาทีนี้ โดยไม่มองไปข้างหน้าหรือย้อนดูอดีต เหมือนคนเดินขึ้นเขาที่เลือกก้าวเท้าไปทางที่ชันที่สุดเสมอ เพราะคิดว่าจะถึงยอดเขาเร็วสุด

🍕 ตัวอย่างง่ายๆ — การทอนเงิน:
ลูกค้าจ่าย 1,000 ซื้อของ 120 บาท (ทอน 880) → เราหยิบแบงก์ 500 ก่อน (ใบใหญ่สุด) → แล้วหยิบแบงก์ 100 (3 ใบ) → แล้วหยิบแบงก์ 50 → แล้วหยิบเหรียญ
เราไม่หยิบเหรียญบาท 880 เหรียญแน่นอน — นี่คือ Greedy!
สถานการณ์Greedy Strategyเรียงลำดับ
ทอนเงินหยิบเหรียญใหญ่สุดก่อนมาก → น้อย
นักช้อปงบน้อยซื้อของถูกที่สุดก่อนน้อย → มาก
ดูหนังให้ได้มากเรื่องเลือกหนังที่จบเร็วสุดก่อนเวลาจบ น้อย → มาก
ให้อาหารแมวจับคู่แมวที่ต้องการน้อยกับปลาเล็กสุดน้อย → มาก
🔑 หัวใจสำคัญคือ "การเรียงลำดับ":
เราจะเป็นคนตะกละได้... เราต้องรู้ก่อนว่าอะไรคือ "ชิ้นที่ดีที่สุด" — Greedy มักจะมาคู่กับ Sorting เสมอ

3 Steps of Greedy

1️⃣
Selection
เลือกชิ้นที่ดีที่สุด ณ ตอนนั้น จากข้อมูลที่ Sort แล้ว
เช่น: หยิบเหรียญที่ใหญ่ที่สุด
2️⃣
Feasibility Check
เช็คเงื่อนไขว่า "เอามาแล้วไม่พังใช่ไหม"
เช่น: หยิบเหรียญ 10 มา แต่เงินทอนเหลือแค่ 5 → หยิบไม่ได้
3️⃣
Solution Update
ถ้าเงื่อนไขผ่าน → เก็บเข้ากระเป๋า (คำตอบ) แล้วไปทำข้อ 1 ใหม่
เช่น: หยิบเหรียญได้ → count++ แล้ว loop ต่อ
⚠️ Greedy ไม่ได้แก้ได้ทุกโจทย์บนโลก!
แต่สำหรับโจทย์ส่วนใหญ่ในระดับเริ่มต้น มันคือท่าไม้ตายที่ง่ายและเร็วที่สุด ถ้า Greedy ใช้ไม่ได้ ต้องใช้ Dynamic Programming (DP)

⚠️ The Trap — เมื่อ Greedy พัง!

สมมติเราไปดาวอังคาร ที่นั่นมีเหรียญแปลกๆ คือ 4 บาท, 3 บาท, 1 บาท — ต้องการทอนเงิน 6 บาท ให้ได้จำนวนเหรียญน้อยที่สุด

❌ Greedy (ผิด)
เห็นเลข 6... รีบคว้าเหรียญ 4 ก่อน (เพราะมากสุด)
เหลือ 2... ต้องใช้เหรียญ 1 + 1
สรุป: ได้ 3 เหรียญ (4+1+1)
✅ Optimal (ถูก)
มองภาพรวม... ไม่หยิบ 4
เลือกหยิบ 3 + 3
สรุป: ได้ 2 เหรียญ เท่านั้น!
Checklist อย่างง่าย:
1. ถ้าโจทย์เป็นระบบมาตรฐาน (เช่น เงินไทย, เงินดอลลาร์) → Greedy มักจะรอด
2. ถ้าโจทย์กำหนดตัวเลขมาแปลกๆ หรือให้ Input อิสระ → ให้ระวัง! Greedy อาจพัง
3. ถ้า Greedy ใช้ไม่ได้ → ต้องใช้ Dynamic Programming (DP)

coin_change(amount) — หาจำนวนเหรียญน้อยที่สุดในการทอนเงิน

เหรียญ [10, 5, 2, 1] มีไม่จำกัดจำนวน · ต้องคืนค่าจำนวนเหรียญทั้งหมดที่ใช้

ตัวอย่าง call
coin_change(18) coin_change(7) coin_change(23)
ผลลัพธ์
4 3 4
เขียนโค้ดex1.py
def coin_change(amount):
  coins = [10, 5, 2, 1]
  total = 0
  for coin in coins:
    total += amount // coin
    amount = amount % coin
  return total

max_items(budget, prices) — หาจำนวนชิ้นสูงสุดที่ซื้อได้

ซื้อให้ได้จำนวนชิ้นมากที่สุดโดยไม่เกิน budget · Greedy: ซื้อของถูกที่สุดก่อน

ตัวอย่าง call

ผลลัพธ์
3 2 0
เขียนโค้ดex2.py
def max_items(budget, prices):
  prices.sort() # เรียงน้อยไปมาก
  count = 0
  for price in prices:
    if budget >= price:
      budget -= price
      count += 1
    else: break
  return count

assign_cookies(greed, size) — หาจำนวนแมวที่อิ่มมากที่สุด

แมวตัวที่ i จะพอใจก็ต่อเมื่อได้ปลาขนาดอย่างน้อย greed[i] · Greedy: Sort ทั้งคู่ แล้วจับคู่ปลาเล็กสุดกับแมวที่ต้องการน้อยสุด

def assign_cookies(greed, size):
  greed.sort()
  size.sort()
  cat = 0 # ตัวชี้แมว
  for s in size:
    if cat < len(greed) and s >= greed[cat]:
      cat += 1
  return cat

max_movies(intervals) — ดูหนังให้ได้มากเรื่องที่สุด

intervals คือ [[start,end],...] · ดูหนังซ้อนกันไม่ได้ · Greedy: เรียงตาม end time แล้วเลือกหนังที่จบเร็วสุดก่อน

ตัวอย่าง call

2 3 2
เขียนโค้ดex4.py
def max_movies(intervals):
  intervals.sort(key=lambda x: x[1])
  count = 0
  last_end = -1
  for start, end in intervals:
    if start >= last_end:
      count += 1
      last_end = end
  return count

def min_cost_trip(costs, k):
  costs.sort() # เรียงน้อยไปมาก
  return sum(costs[:k]) # เอา k ตัวแรก

def pack_bag(capacity, weights):
  weights.sort()
  count = 0
  for w in weights:
    if capacity >= w:
      capacity -= w
      count += 1
    else: break

min_jobs(tasks, time_limit) — ทำงานให้ได้มากชิ้นภายในเวลาที่กำหนด

tasks คือลิสต์เวลาที่ใช้ต่อชิ้น (นาที) · ต้องการทำให้ได้มากชิ้นภายใน time_limit นาที · Greedy: เรียงเวลาน้อยไปมาก แล้วเลือกงานที่ใช้เวลาน้อยสุดก่อน

ตัวอย่าง call

ผลลัพธ์
4 2 3
เขียนโค้ดex7.py
def min_jobs(tasks, time_limit):
  tasks.sort()
  count = 0
  time_used = 0
  for t in tasks:
    if time_used + t <= time_limit:
      time_used += t
      count += 1
    else: break
  return count

แบ่งขนม

8

share_candy(candies, n) — แบ่งขนมให้เด็ก n คน คนละเท่าๆ กัน ได้มากสุด

candies คือจำนวนขนมแต่ละถุง · แบ่งขนมให้เด็ก n คน โดยแต่ละคนต้องได้เท่ากัน คืนจำนวนขนมสูงสุดต่อคน · Greedy: Binary search หรือ sum แล้วหารด้วย n

ตัวอย่าง call
share_candy([5,3,2], 3) share_candy([10,2,3], 2) share_candy([1,2], 7)
ผลลัพธ์
3 7 0
💡 แนวคิด: ขนมรวมทั้งหมด = sum(candies) · แต่ละคนได้ไม่เกิน sum // n · แต่ต้องตรวจว่าแต่ละถุงสามารถแบ่งได้ตามจำนวนที่ต้องการด้วย (นับว่าแต่ละถุงส่งได้กี่ส่วนๆ ละ k ชิ้น)
เขียนโค้ดex8.py
def share_candy(candies, n):
  lo, hi = 0, sum(candies) // n
  while lo < hi:
    mid = (lo + hi + 1) // 2
    if sum(c // mid for c in candies) >= n:
      lo = mid
    else:
      hi = mid - 1
  return lo

min_remove(intervals) — หาจำนวน interval น้อยสุดที่ต้องลบออกเพื่อไม่ให้ซ้อนทับกัน

คล้าย Activity Selection แต่คืนจำนวนที่ต้อง "ตัดออก" · Greedy: นับ interval สูงสุดที่ไม่ซ้อนทับได้ แล้วเอาทั้งหมดลบออก

1 2 0
เขียนโค้ดex10.py
def min_remove(intervals):
  intervals.sort(key=lambda x: x[1])
  keep = 0
  last_end = float('-inf')
  for start, end in intervals:
    if start >= last_end:
      keep += 1
      last_end = end
  return len(intervals) - keep
