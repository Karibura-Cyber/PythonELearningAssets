Chapter 01

Introduction to
Object-Oriented Programming

เรียนรู้แนวคิดพื้นฐานของ OOP ที่เป็นหัวใจสำคัญของการเขียนโปรแกรมสมัยใหม่

OOP คืออะไร?

Object-Oriented Programming (OOP) คือ กระบวนทัศน์การเขียนโปรแกรม (Programming Paradigm) ที่จัดระเบียบโค้ดรอบ ๆ "วัตถุ" (Objects) แทนที่จะเป็นแค่ฟังก์ชันและลอจิก วัตถุเหล่านี้เป็นตัวแทนของสิ่งต่าง ๆ ในโลกจริง โดยมีทั้ง คุณสมบัติ (Attributes) และ พฤติกรรม (Methods)

🧩 เปรียบเทียบ Paradigm
Procedural
name = "สมชาย"
balance = 1000

def deposit(amount):
    global balance
    balance += amount

def withdraw(amount):
    global balance
    balance -= amount

⚠️ ข้อมูลกระจัดกระจาย จัดการยาก

→
OOP
class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount

    def withdraw(self, amount):
        self.balance -= amount

✅ ข้อมูล + พฤติกรรมอยู่ด้วยกัน

Chapter 02

Classes &
Objects

Class คือพิมพ์เขียว (Blueprint) และ Object คือสิ่งที่สร้างจากพิมพ์เขียวนั้น

Class = Blueprint, Object = Instance

ลองนึกภาพ Class เหมือน แม่พิมพ์ขนม — แม่พิมพ์หนึ่งอันสามารถทำขนมได้หลายชิ้น แต่ละชิ้นอาจมีรสชาติต่างกัน แต่มีรูปทรงเดียวกัน

📐 โครงสร้าง Class
class ClassName:
    """Docstring อธิบายคลาส"""
    
    # Class Attribute (แชร์ร่วมกันทุก instance)
    class_variable = value
    
    # Constructor (เรียกตอนสร้าง object)
    def __init__(self, param1, param2):
        self.param1 = param1      # Instance Attribute
        self.param2 = param2
    
    # Instance Method
    def method_name(self):
        return self.param1
    
    # String Representation
    def __str__(self):
        return f"ClassName({self.param1})"

Naming Conventions

Class Name PascalCase — เช่น BankAccount, StudentRecord
Method/Attribute snake_case — เช่น get_balance(), first_name
Private _underscore_prefix — เช่น _secret, __very_private
Constant UPPER_SNAKE_CASE — เช่น MAX_SIZE, PI

Chapter 03

Constructor
__init__ Method

เมธอดพิเศษที่ถูกเรียกโดยอัตโนมัติเมื่อสร้าง Object ใหม่

__init__ คืออะไร?

__init__ คือ Constructor ของ Python — มันถูกเรียกอัตโนมัติทุกครั้งที่สร้าง object ใหม่จาก class ใช้สำหรับตั้งค่าเริ่มต้น (Initialize) ให้กับ attributes ต่าง ๆ ของ object

💡
จริง ๆ แล้ว __new__ คือ method ที่สร้าง object จริง ๆ ส่วน __init__ แค่ตั้งค่าเริ่มต้นให้มัน แต่ในทางปฏิบัติ เราเรียก __init__ ว่า constructor เพราะมันคือที่ที่เรากำหนดค่าเริ่มต้น

Chapter 04

Methods &
Self Parameter

เข้าใจ self และประเภทต่าง ๆ ของ Methods ใน Python Class

self คืออะไร?

self คือ ตัวแทนของ object ปัจจุบัน ที่กำลังเรียกใช้ method เมื่อเราเขียน dog1.bark() Python จะแปลงเป็น Dog.bark(dog1) โดยอัตโนมัติ โดย dog1 จะถูกส่งเข้าไปเป็น self

🔄 self ทำงานอย่างไร
# เมื่อเราเขียน:
dog1.bark()

# Python ทำงานเบื้องหลังเหมือน:
Dog.bark(dog1)     # dog1 ถูกส่งเป็น self

# ดังนั้นภายใน method:
def bark(self):     # self = dog1
    return self.name  # = dog1.name

Chapter 06

Encapsulation
การห่อหุ้มข้อมูล

ปกป้องข้อมูลภายใน Object ด้วย Access Modifiers และ Properties

Access Levels ใน Python

public self.name — เข้าถึงได้จากทุกที่
_protected self._name — Convention บอกว่า "ไม่ควรเข้าถึงจากภายนอก" (แต่ยังทำได้)
__private self.__name — Name Mangling → กลายเป็น _ClassName__name
⚠️
Python ไม่มี true private เหมือน Java/C++ การใช้ __ แค่ทำ name mangling ไม่ได้ป้องกันจริง ๆ แต่เป็น convention ที่บอกว่า "อย่าเข้าถึงโดยตรง"

การสืบทอด (Inheritance) ช่วยให้เราสร้าง Child Class ที่รับคุณสมบัติและเมธอดมาจาก Parent Class โดยสามารถเพิ่มหรือแก้ไขพฤติกรรมได้

        /      \
      Dog      Cat        ← Child (Derived / Sub) Class
