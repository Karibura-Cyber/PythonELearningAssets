Introduction

ทำไมต้องเลิกใช้ List ในบางสถานการณ์?

การหาของใน List N ตัว ต้องวิ่งผ่านทุกตัว ใช้เวลา O(N) แต่ Dictionary และ Set ใช้ Hash Table ค้นหาได้ O(1) เร็วคงที่เสมอ

List Search
O(N)
ช้าลงตามจำนวนข้อมูล
Set Lookup
O(1)
เร็วคงที่ทุกกรณี
Dict Lookup
O(1)
เร็วคงที่ทุกกรณี
ในข้อสอบ TOI Zero / สอวน. จำกัดเวลา 1 วินาที — O(N²) มักสอบตก (TLE) จึงต้องใช้ O(1)

Set Operations — คณิตศาสตร์ของเซต

A | B
Union — รวมทั้งหมด
รวมสมาชิกจากทั้งสองเซต
{1,2,3} | {3,4,5} → {1,2,3,4,5}
A & B
Intersection — ส่วนเหมือนกัน
เอาเฉพาะที่มีในทั้งสองเซต
{1,2,3} & {3,4,5} → {3}
A - B
Difference — ส่วนต่าง
เอาที่มีใน A แต่ไม่มีใน B
{1,2,3} - {3,4,5} → {1,2}
A ^ B
Sym. Difference
เอาที่ไม่ซ้ำกันของทั้งสองเซต
{1,2,3} ^ {3,4,5} → {1,2,4,5}

Dictionary — เก็บข้อมูลแบบ Key: Value

Dictionary คือโครงสร้างข้อมูลที่เก็บข้อมูลเป็นคู่ Key : Value เปรียบเหมือนสมุดโทรศัพท์ที่ค้นหาชื่อแล้วได้เบอร์ทันที

กฎเหล็ก 3 ข้อ:
1. Key ห้ามซ้ำ — คอมฯ จะสับสนว่าจะดึงค่าไหน
2. Value ซ้ำได้ — คนสองคนมีคะแนนเท่ากันได้
3. Mutable — เปลี่ยนแปลงค่าข้างในได้ตลอด

Part 4

Why Mix Them? — ใช้ Set + Dictionary ร่วมกัน

  • Set → ใช้สำหรับ "ตรวจสอบเงื่อนไข" หรือ "หาจุดร่วม/จุดต่าง"
  • Dictionary → ใช้สำหรับ "เก็บสถานะ" หรือ "ประมวลผลข้อมูล"
  • Efficiency → ผสมทั้งสองหลีกเลี่ยง Loop ซ้อน Loop O(N²) → ผ่าน Time Limit
