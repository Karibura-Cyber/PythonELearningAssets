Part 2

Python มีของดีให้แล้ว — .sort() vs sorted()

Python ใช้ Algorithm ลับชื่อ "Timsort" (ผสมระหว่าง Merge Sort + Insertion Sort) เร็วระดับ O(n log n)

✏️
.sort()
เปลี่ยนแปลงข้อมูลใน List เดิม ทันที (In-Place) — ไม่คืนค่าอะไรกลับมา (return None)
ประหยัดเมมโมรี่ — ใช้เมื่อไม่ต้องการข้อมูลดิบ
🆕
sorted()
สร้าง List ใหม่ที่เรียงแล้ว (ตัวเก่าเหมือนเดิมเป๊ะ) — คืนค่า List ใหม่กลับมา
ใช้เมื่อยังอยากเก็บข้อมูลดิบไว้เทียบดู
ลองรันsort_vs_sorted.py
Output
กด "รันโค้ด" เพื่อดูผลลัพธ์

จุดสำคัญ: .sort() คืนค่า None!

Bug พบบ่อยมาก! ระวัง result = data.sort() — result จะได้เป็น None เพราะ .sort() แก้ in-place แล้วคืน None. ถ้าอยากเก็บใส่ตัวแปร ให้ใช้ result = sorted(data) แทน!
คำสั่งแก้ตัวเก่าreturn valueเหมาะกับ
.sort()✅ ใช่ (In-Place)Noneไม่ต้องการข้อมูลเก่า, ประหยัด RAM
sorted()❌ ไม่แตะList ใหม่ต้องเก็บข้อมูลดิบไว้

The Power of Key & Lambda Function

สั่งให้คอมพิวเตอร์เรียงข้อมูล "ตามกฎของเรา" — เมื่อ A-Z ไม่ใช่คำตอบของทุกเรื่อง

Lambda Function คือฟังก์ชันสั้นๆ บรรทัดเดียว ที่ไม่มีชื่อ (Anonymous Function) เอาไว้ใช้เฉพาะกิจ

def get_len(x): return len(x)  ≡  lambda x: len(x)

เทคนิค Custom Sort

KeySort by Length
เรียงตามความยาวคำ — key=lambda x: len(x)
KeyCase-Insensitive
ไม่สนตัวพิมพ์เล็ก/ใหญ่ — key=lambda x: x.lower()
KeySort by Field
เรียงใน Tuple/List — key=lambda x: x[1] สำหรับ field ที่ 2
AdvancedMulti-Key Sort
เรียงหลาย criteria — key=lambda x: (x[1], x[0])
