Part 3 — ⚠️ ระวัง!

O(n²) — Quadratic Time

เมื่อเราใส่ Loop ซ้อนไว้ใน Loop (Nested Loop) ความเหนื่อยจะไม่ได้เพิ่มแบบบวก แต่จะเพิ่มแบบคูณ

  • Loop นอกรัน n รอบ และ Loop ในรัน n รอบ → รวมทำงาน n × n = n² ครั้ง
  • ข้อมูลแค่ 100 ตัว → คอมต้องทำงานถึง 10,000 ครั้ง!
  • ข้อมูล 1,000 ตัว → ทำงาน 1,000,000 ครั้ง!
กฎง่ายๆ: ถ้าเห็น for ซ้อน for และทั้งคู่วนตาม n → นั่นคือ O(n²) ต้องระวัง!

O(log n) คือจำนวนครั้งที่เราสามารถ "หาร 2" ไปได้เรื่อยๆ จนเหลือ 1

นี่คือพลังของ Binary Search! ข้อมูล 1 ล้านตัว แต่หาเจอได้ใน 20 ครั้ง ขณะที่ Linear Search ต้องใช้ถึง 1,000,000 ครั้ง
