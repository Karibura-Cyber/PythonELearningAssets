Top-down vs Bottom-up — ต่างกันยังไง?

🔽
Top-down (Week 9)
เริ่มจากโจทย์ใหญ่ (N) → แตกเป็นโจทย์ย่อย ใช้ Recursion + Memoization (สมุดจด)
❗ ข้อจำกัด: N ใหญ่มาก → Recursion Limit Exceeded
🔼
Bottom-up (สัปดาห์นี้)
ไม่เรียกซ้อน แต่ "วนลูป" สร้างคำตอบจากเล็กไปใหญ่ เหมือนก่อกำแพงอิฐจากฐานขึ้นบน
✅ เปลี่ยนจาก "สมุดจด" เป็น "ตาราง" (List/Array)

มีเหรียญ 1, 3, 4 บาท ต้องการทอนเงิน 6 บาท ใช้เหรียญน้อยที่สุดกี่เหรียญ

Min Cost Climbing Stairs — ขึ้นบันไดแบบประหยัดพลังงาน

บันไดแต่ละขั้นเสียพลังงานต่างกัน เริ่มที่ขั้น 0 หรือ 1 ก็ได้ จงหาพลังงานรวมที่น้อยที่สุดเพื่อขึ้นไปถึงดาดฟ้า

สูตร: dp[i] = cost[i] + min(dp[i-1], dp[i-2])
คำตอบ = min(dp[-1], dp[-2]) (เลือกขาออกที่ถูกที่สุด)

Grid Traveler

หุ่นยนต์เดินในตาราง — DP 2 มิติ

ตารางขนาด M×N หุ่นยนต์เริ่มมุมซ้ายบน เดินได้แค่ "ขวา" กับ "ลง" ไปให้ถึงมุมขวาล่าง มีกี่วิธี

สูตร 2 มิติ: dp[r][c] = dp[r-1][c] + dp[r][c-1]
Base Case: แถวแรก = 1 ทั้งแถว, คอลัมน์แรก = 1 ทั้งคอลัมน์

เขียนโค้ดex1.py
dp = [0]*(n+1)
if n==0: return 0
dp[1]=1
for i in range(2,n+1): dp[i]=dp[i-1]+dp[i-2]
return dp[n]

จากนั้น dp = [0]*(n+1); dp[1]=1; dp[2]=2
for i in range(3,n+1): dp[i]=dp[i-1]+dp[i-2]
return dp[n]

Coin Change

coin_change(coins, amount) — เหรียญน้อยที่สุดในการทอน

return จำนวนเหรียญน้อยที่สุด ถ้าทอนไม่ได้ให้ return -1

call
coin_change([1,3,4], 6) coin_change([2], 3) coin_change([1,5,6,9],11)
output
2 -1 2
เขียนโค้ดex3.py
dp = [float('inf')]*(amount+1); dp[0]=0
for i in range(1,amount+1):
  for c in coins:
    if i-c >= 0: dp[i]=min(dp[i],dp[i-c]+1)
return dp[amount] if dp[amount]!=float('inf') else -1

เขียนโค้ดex4.py
n=len(cost); dp=cost[:]
for i in range(2,n): dp[i]+=min(dp[i-1],dp[i-2])
return min(dp[-1],dp[-2])

เขียนโค้ดex5.py
dp=[[0]*n for _ in range(m)]
for r in range(m): dp[r][0]=1
for c in range(n): dp[0][c]=1
for r in range(1,m):
  for c in range(1,n): dp[r][c]=dp[r-1][c]+dp[r][c-1]
return dp[m-1][n-1]
